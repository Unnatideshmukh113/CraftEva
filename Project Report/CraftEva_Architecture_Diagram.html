✏️ How to Draw This on a Whiteboard

Draw 4 vertical boxes left → right. Total time: ~90 seconds to sketch.

① FRAME

Draw the 4 main boxes

WHAT TO DRAW

Draw 4 large equal rectangles across the board, side by side. Label them: FRONTEND (blue), SECURITY (pink), BACKEND (green), DATABASE (yellow). Add a horizontal arrow pointing right across all boxes at the top labelled "HTTP Request flow →".

WHAT TO SAY
"CraftEva follows a classic multi-tier architecture — React frontend, Spring Security layer, Spring Boot backend, and MySQL database, all communicating over HTTP with JWT tokens."
② FRONTEND

Fill the React layer

WHAT TO DRAW

Inside FRONTEND box: draw a browser mockup outline. List pages: Home, Login, Register, Cart, Payment, Orders, SellerDashboard, AdminDashboard. Add a dark box labelled "Axios — Bearer Token" and a box labelled "AuthContext → localStorage".

WHAT TO SAY
"The React frontend stores the JWT in localStorage, attaches it to every Axios request, and uses AuthContext to share auth state. Routes are protected based on the role decoded from the token."
③ SECURITY

Draw the JWT chain

WHAT TO DRAW

Inside SECURITY box: draw a thick dark box "JwtAuthenticationFilter → OncePerRequestFilter". Below it: "JwtUtil: HMAC-SHA256, claims={email, role}". Add a small red box "401" (no token) and purple box "403" (wrong role). Write "BCrypt.matches()" in a separate small box.

WHAT TO SAY
"Every request goes through JwtAuthenticationFilter first. It extracts the Bearer token, validates the signature and expiry with JwtUtil, reads the role claim, and sets the SecurityContextHolder — only then does it reach any controller."
④ BACKEND

Draw the 3-column MVC

WHAT TO DRAW

Split the BACKEND box into 3 narrow columns with dashed vertical lines: @Controller | @Service | @Repository. List controllers in column 1, services in column 2, repositories in column 3. Add a bracket on OrderServiceImpl labelled "@Transactional".

WHAT TO SAY
"The backend uses standard MVC — thin controllers, fat services, repository layer using Spring Data JPA. Services are marked @Transactional wherever we do multiple database writes — like placeOrder which saves the order and links order items atomically."
⑤ DATABASE

Draw the 5 tables

WHAT TO DRAW

Draw a database cylinder at the top. Below it, draw 5 stacked cards: users, products (FK: seller_id→users), orders (FK: buyer_id→users), orderItems (FK: product_id, order_id), payment (FK: order_id [1:1], seller_id). Mark PKs in gold, FKs in blue.

WHAT TO SAY
"We have 5 core tables. Users is a single table with a role column — one table for buyers, sellers, and admins. payment has a 1-to-1 with orders, and orderItems is the junction table between orders and products."
⑥ CONNECT

Draw arrows & number the journey

WHAT TO DRAW

Draw solid arrows → between boxes labelled: ① "HTTP + Bearer JWT" → ② "validateToken()" → ③ "DTO + Service call" → ④ "SQL". Then draw a long curved dashed arrow ← along the bottom labelled "Response: JSON + HTTP Status (200/201/401/403/404/500)".

WHAT TO SAY
"The request flows left to right through all four layers, and the response flows back as JSON with the appropriate HTTP status code — 200 for success, 201 for resource created, 401 when the token is missing or expired, 403 when the role is wrong."

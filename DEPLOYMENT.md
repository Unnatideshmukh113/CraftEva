# 🚀 CraftEva Deployment Guide

This guide covers deploying CraftEva to production. The recommended setup is:
- **Backend** → [Render](https://render.com) (free tier available)
- **Frontend** → [Vercel](https://vercel.com) or [Netlify](https://netlify.com) (free tier)
- **Database** → [Aiven](https://aiven.io) or [PlanetScale](https://planetscale.com) (free MySQL cloud tier)

---

## 📋 Prerequisites

- [ ] GitHub account with the CraftEva repository
- [ ] Render.com account
- [ ] Vercel or Netlify account
- [ ] Cloud MySQL provider account (Aiven / PlanetScale / Railway)
- [ ] Git installed locally

---

## 🗄️ Step 1: Set Up Cloud Database (MySQL)

### Option A: Aiven (Recommended)

1. Go to [https://aiven.io](https://aiven.io) and sign up
2. Create a new **MySQL** service (free tier → select closest region)
3. Once provisioned, note the connection details:
   ```
   Host: your-service.aivencloud.com
   Port: 3306
   Database: defaultdb
   Username: avnadmin
   Password: <auto-generated>
   ```
4. Navigate to **Databases** tab → create a database named `crafteva_db`
5. Download the **CA certificate** if SSL is required

### Option B: Railway

1. Go to [https://railway.app](https://railway.app) and sign up
2. Click **New Project** → **Provision MySQL**
3. In the **Variables** tab, note:
   - `MYSQL_URL`, `MYSQL_USER`, `MYSQL_PASSWORD`, `MYSQL_DATABASE`

---

## ⚙️ Step 2: Configure Backend Environment Variables

The backend requires the following environment variables in production. **Never commit secrets to Git.**

| Variable | Description | Example |
|---|---|---|
| `SPRING_DATASOURCE_URL` | Full JDBC URL | `jdbc:mysql://host:3306/crafteva_db?useSSL=true` |
| `SPRING_DATASOURCE_USERNAME` | DB username | `avnadmin` |
| `SPRING_DATASOURCE_PASSWORD` | DB password | `your_password` |
| `JWT_SECRET` | JWT signing key (min 256-bit) | `a-very-long-random-secret-key-32chars+` |
| `JWT_EXPIRATION` | Token expiry in ms | `86400000` (24 hours) |
| `SERVER_PORT` | Server port | `8080` |

## ☁️ Step 3: Deploy Backend to Render

### 3.1 Build the JAR locally (verify it works first)

```bash
cd crafteva-backend/springboot_backend_template
./mvnw clean package -DskipTests
# JAR will be at: target/springboot_backend_template-0.0.1.jar
```

### 3.2 Create a Render Web Service

1. Log in to [https://render.com](https://render.com)
2. Click **New** → **Web Service**
3. Connect your GitHub account and select the `CraftEva` repository
4. Configure the service:

   | Setting | Value |
   |---|---|
   | **Name** | `crafteva-backend` |
   | **Region** | Closest to your users |
   | **Branch** | `main` |
   | **Root Directory** | `crafteva-backend/springboot_backend_template` |
   | **Runtime** | `Java` |
   | **Build Command** | `./mvnw clean package -DskipTests` |
   | **Start Command** | `java -jar target/springboot_backend_template-0.0.1.jar` |

5. Under **Environment Variables**, add all variables from Step 2
6. Click **Create Web Service**
7. Render will build and deploy (takes ~5 minutes on first deploy)

### 3.3 Verify Backend Deployment

Once deployed, your backend URL will be:
```
https://crafteva-backend.onrender.com
```

Test it:
```bash
curl https://crafteva-backend.onrender.com/actuator/health
# Expected: {"status":"UP"}
```

Swagger UI:
```
https://crafteva-backend.onrender.com/swagger-ui/index.html
```

## 🌐 Step 4: Deploy Frontend to Vercel

### 4.1 Configure Frontend Environment

Create `crafteva_frontend/vite-project/.env.production`:

```env
VITE_API_BASE_URL=https://crafteva-backend.onrender.com
```

### 4.2 Deploy to Vercel

**Option A: Vercel CLI**

```bash
npm install -g vercel
cd crafteva_frontend/vite-project
npm install
vercel --prod
```

**Option B: Vercel Dashboard**

1. Go to [https://vercel.com](https://vercel.com) and sign up
2. Click **Add New Project** → import your GitHub `CraftEva` repo
3. Configure:

   | Setting | Value |
   |---|---|
   | **Framework Preset** | Vite |
   | **Root Directory** | `crafteva_frontend/vite-project` |
   | **Build Command** | `npm run build` |
   | **Output Directory** | `dist` |

4. Under **Environment Variables**, add:
   ```
   VITE_API_BASE_URL = https://crafteva-backend.onrender.com
   ```
5. Click **Deploy**

### 4.3 Verify Frontend Deployment

Your frontend URL will be:
```
https://crafteva.vercel.app
```

## 🔄 Step 5: Alternative — Deploy Frontend to Netlify

1. Go to [https://app.netlify.com](https://app.netlify.com)
2. Click **Add new site** → **Import an existing project**
3. Connect GitHub → select `CraftEva`
4. Configure:

   | Setting | Value |
   |---|---|
   | **Base directory** | `crafteva_frontend/vite-project` |
   | **Build command** | `npm run build` |
   | **Publish directory** | `crafteva_frontend/vite-project/dist` |

5. Under **Site settings** → **Environment variables**, add:
   ```
   VITE_API_BASE_URL = https://crafteva-backend.onrender.com
   ```
6. Click **Deploy**

> ⚠️ Add a `_redirects` file inside `crafteva_frontend/vite-project/public/` for React Router to work on Netlify:
> ```
> /*    /index.html   200
> ```

## 🔍 Troubleshooting

### Backend won't start on Render
- Check **Logs** tab on Render for error details
- Ensure all environment variables are set correctly
- Verify the database connection string and credentials
- Make sure the DB allows external connections (check firewall/whitelist Render's IPs)

### Frontend shows CORS error
- Ensure `VITE_API_BASE_URL` points to the correct Render backend URL
- Check that the Spring Boot backend has CORS configured to allow your frontend domain
- Update `config/` in the backend to whitelist your Vercel/Netlify URL

### Database connection refused
- Check that the database host allows connections from external IPs
- For Aiven: ensure the correct SSL certificate is used if SSL is enforced
- For Railway: double-check the `MYSQL_URL` format

### JWT errors after deployment
- Ensure `JWT_SECRET` is at least 32 characters long
- Verify the secret is identical between deployments (not regenerated)
